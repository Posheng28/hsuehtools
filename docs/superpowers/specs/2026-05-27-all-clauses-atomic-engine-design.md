# 注意款全補齊 + 原子條件引擎 設計（2026-05-27）

## 背景與目標

DisposalTool 目前只判 款一①②、款二(純價格)、款三。實際注意用到 款一~十二。本案：
1. 把判定重構成「**原子條件引擎**」——每款一個獨立純函式評估器，回該款是否成立；組合層彙整成 `{first, any}` 餵處置規則。利於正確性、UI 獨立顯示、未來回測共用。
2. **補齊可行款**：款二補完整（差幅80%）、款六(PE/PBR)、款十一(起迄價差)、款十二(借券)。
3. **修 trunc2 浮點精度**（47.3/43.0 應為 10.00 非 9.99）。
4. 款四(週轉率)/款五(券商分點)**資料卡死**，UI 標「資料不足」、不判定。

錨定：2026-05-26 上櫃注意（窗口基準 5/18 → 6 間隔 5/19~5/26 含當日）。所有公式已對拍實際注意值。

## 已驗證事實（回推 attstock + 官方注意值）

- **款一**：6 日累積 = `Σ trunc2(逐日%)`（6 間隔）。3581=27.47、6182=44.72、5321=59.47、4127=48.30 全中。門檻 上櫃 ①>30% / ②>23%+起迄價差≥40 + 差幅≥20%（上市 32/25/50）。
- **款二**：起迄 = `close(當日)/close(窗口第一天)−1`（比值，非最低收盤）。波若威90日=219.37、金山電30日=126.15 全中。門檻 上櫃 30/60/90日 >100/140/160%，差幅≥**80%**（上市 100/130/160）。全體基底 = 加權指數 30/60/90日（上櫃櫃買指數；上市等權，與款一一致）。
- **款十一**：`|close(當日) − close(窗口第一天)|` ≥ gap。弘塑535/順達110/茂達87.5/勤凱92 全中。上櫃 gap 基底70（收盤≥300 起每+300 加級距+15）；上市 100（每+500+25）。
- **款六**：個股 PE/PBR 來自 `tpex.org.tw/openapi/v1/tpex_mainboard_peratio_analysis?date=YYYY/MM/DD`（欄位 `PriceEarningRatio`/`PriceBookRatio`）；波若威 PE141.20/PBR30.78 命中。門檻 上櫃 PE≥65且>全體加權均×2 或 PBR≥4且>均×2（上市 60/6）。
- **款十二**：借券來自 `tpex.org.tw/www/zh-tw/margin/sbl?date=YYYY/MM/DD&response=json`（借券當日賣出）；4128=10.39%/4174=11.21% 命中。門檻 上櫃 6日借券賣出率>9% 且 放大≥4×60日均量（上市 12%/5×）。
- **全體均值算法**（沿用已上線）：上市=普通股逐日截斷相加等權平均；上櫃=櫃買指數逐日相加。30/60/90 日同理。

## 架構：原子條件引擎

新增 `lib/clauseEngine.ts`（純函式，無 React/IO 依賴），核心：

```
type ClauseInput = {
  market: Market
  closes: number[]          // 統一收盤路徑(基準..計算日)，計算日為待測價
  i: number                 // 卡索引（窗口位置）
  marketAvg6: number|null    // 全體 6日累積%(款一差幅基底)
  peRatio/pbRatio: number|null; mktAvgPE/PBR: number|null   // 款六
  sblRate6: number|null; sblAmp: number|null                // 款十二
  volRatio: number|null      // 款三(當日量/60日均量)
  assumeDaily: boolean       // 「假設當日量/籌碼達標」開關
}
type ClauseResult = { id: '1①'|'1②'|'2'|'3'|'6'|'11'|'12'; fired: boolean; first: boolean; detail: string; blocked?: boolean }
function evalClauses(input): ClauseResult[]
```

- 每款一個 `evalClauseN(input): ClauseResult`，彼此獨立、可單測。
- 組合：`first = any(r.first)`、`any = any(r.fired)` → 餵 `computeTriggers`（沿用，介面改吃 `{first,any}[]`）。
- DisposalTool 只負責「組裝 input + 呈現」；判定全在引擎。利於回測直接重用引擎。

## 各款評估器規格

### 款一①②（已有，移入引擎）
`累積 = Σ trunc2(逐日%)`(基準→計算日前一日 = knownSum) + `trunc2(計算日當日%)`。門檻價 `prevClose×(1+(max(門檻%, 全體6日+20)−knownSum)/100)` nextTick；款一② 另 `clTick(spreadBase+gap價差40)`。`first=true`。

### 款二（移入引擎，維持純價格）
`pct_N = close(計算日)/close(窗口第一天)−1`（比值，N=30/60/90，由短到長取最先達標）。**達標 = pct_N > 門檻%**（上櫃 100/140/160）。沿用防重複豁免（30日內有款一注意 且 6日累積 ≤ dupPct 27%上櫃 → 不適用）。`first=false`、`any=true`。
**差幅≥80% 不計算**（使用者決定）：30/60/90 日漲幅動輒 >100%，差幅(vs 全體~20%) 幾乎必 ≥80%、非 binding gate；故**不抓全體 30/60/90**，款二視同差幅已達。起迄定義已驗證（波若威90日=219.37、金山電30日=126.15 = first→當日）。

### 款三（已有，移入引擎；量差幅略過）
6日累積 > 27%(上櫃)/25%(上市) + 差幅≥20% + 當日量 ≥ 5×60日均量。`assumeDaily` 控當日量。「量差幅≥4×全市場」資料不足、略過並標註。

### 款六（新，下方面板）
`PE' = 預測價 × (現PE/現收盤)`、`PBR'` 同理（隨滑桿變）。達標 = `PE'≥65(上市60) 且 PE'>mktAvgPE×2` 或 `PBR'≥4(上市6) 且 PBR'>mktAvgPBR×2`。次要條件(週轉/券商)資料不足 → `assumeDaily` 開關才計入 any。面板顯示「PE/PBR 達標，但需當日週轉率≥5%或券商集中（無資料，假設）」。

### 款十一（新，卡片）
`價差 = |預測價 − spreadBaseOf(i)|`（spreadBase 已是窗口第一天收盤）。`gap(P) = 上櫃: 70 + 15×級距(P)；上市: 100 + 25×級距(P)`，級距(P)：收盤<300→0，否則每滿 300(上市500)加一級。**確切級距邊界（floor/ceil、含不含）於計畫階段用 attstock `std11.priceDiffThreshold` + 法規原文釘死**。卡片加 `t11 = spreadBase + gap(預測價)` 門檻線。`any=true`。

### 款十二（新，下方面板）
`6日借券率 = Σ6日借券賣出 / Σ6日成交量`、`放大 = 當日借券賣出 / 60日均借券`。達標 = `率>9%(上市12) 且 放大≥4×(上市5)`。當日借券未知 → `assumeDaily`。面板顯示已知 5 日借券率 + 假設開關。

## 新 API 端點
- `app/api/peratio/route.ts`：`?code=&date=` → 個股 PE/PBR + 全體加權均 PE/PBR。上櫃 `tpex_mainboard_peratio_analysis`；上市 TWSE `BWIBBU`(個股本益比/殖利率/股價淨值比)。加權均：以發行股數×收盤為權重 group 全市場（普通股）。快取 6h。
- `app/api/sbl/route.ts`：`?code=&date=` → 窗口 6 日借券賣出量 + 成交量 + 60日均借券。上櫃 `margin/sbl`；上市 TWSE 借券端點（計畫階段確認）。快取 6h。
- **款六/十二 上市+上櫃都接**（兩市場各自的官方端點）。
- `app/api/market-avg`：**不擴充**（款二差幅不做，見款二段）。維持現狀（6 日：上市等權／上櫃櫃買指數）。

## trunc2 精度修正
`const trunc2 = (x:number) => { const v = Math.round(x*1e8)/1e8; return Math.trunc(v*100)/100 }`。先四捨到 8 位殺浮點雜訊再截斷向零。套用：DisposalTool `knownSumOf`、market-avg `twseEqAvg`。驗收：(47.3/43-1)*100 → 10.00（非 9.99）。

## 處置計數（沿用語意）
`first`(款一) 計規則①(連3日)；`any`(任一款) 計規則②③④(連5日/10日6/30日12)。歷史注意 pastNotices：含第一款字樣=level1(first)、其餘=level2(any)。款二/六/十一/十二 模擬日 → level2(any-not-first)。
