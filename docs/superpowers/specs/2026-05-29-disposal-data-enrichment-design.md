# 處置面板資料強化設計

> 日期：2026-05-29　範圍：DisposalTool 的資料面三項強化（自算等權同類/全體均值、接進款一①②差幅、盤中即時卡片）+ 移除非處置資料。
> 前置：方法學已對拍 attstock 新版逐項吻合（見下「驗證」）。

## 背景與目標

處置面板（`components/DisposalTool.tsx`）目前判定「款一①②累積漲跌幅異常」時，差幅閘只用估算/指數法的全體均值，**同類差幅那半邊未實作**；卡片 0（下一交易日預測）整天以「當日 0%」靜態呈現。

本次目標：

1. **自算「同類均值 / 全體均值」**：用該類股下每一檔個股的累積**等權平均**（非加權類股指數），上市/上櫃皆可，純官方資料源，attstock 僅作對拍 oracle。
2. **接進款一①②差幅判定**：差幅 = 標的累積 − 均值，全體與同類兩條都顯示與判定。
3. **盤中即時卡片**：交易日盤中，卡片 0 改為依今日市場即時計算，固定 7 個時點更新。
4. **移除非處置資料**：殖利率/股利相關拿掉；PE/PBR 保留（款六）。

## 驗證（方法學已鎖定 = attstock 新版）

以國巨 2327（上市，產業別 28 電子零組件）對拍，純官方 TWSE 資料自算 = attstock 截圖逐項吻合：

| 項目 | 自算 | attstock 截圖 |
|---|---|---|
| 國巨 已知5日累積(5/22~28) | 27.36% | 27.36%（5日） |
| 同類均值(28類·排除國巨, 5日) | 6.15% → 差幅 21.21% | 6.15%，差幅 21.2% |
| 全體均值(排除國巨, 5日) | 2.22% → 差幅 25.14% | 2.22%，差幅 25.1% |
| 國巨 6日累積(加今日5/29) | 26.83% | =27.36 +今日(−0.53) |

附帶：同類均值加上今日 6 日(5/22~29, 排除國巨) = **8.95%**；國巨 6 日差幅 vs 同類 = 26.83 − 8.95 = 17.88%（今日類股齊漲把同類拉高，6 日基礎的同類差幅已 <20%）。

**鎖定口徑**：
- `22 25 26 27 28` 是「日期」（5/22,25,26,27,28），= 已知 5 日；加今日 = 6 日 = 處置標準。
- 同類均值 = 標的所屬產業別**全部個股**各自累積後**等權平均，排除標的本身**。
- 全體均值 = 全市場個股累積等權平均，**同樣排除標的本身**。
- 每檔每日% = signed(漲跌價差)/前收×100、trunc2、逐日相加（非連乘）；窗口須全期 present 才納入。

## 架構

### 一、新 API：`app/api/sectoravg/route.ts`

輸入 `code`、`market`（上市/上櫃）、基準日（預設今日）。輸出：

```ts
{
  targetCum5: number; targetCum6: number;   // 標的 5日/6日累積
  sectorAvg5: number; sectorAvg6: number;    // 同類均值 5日/6日（排除標的）
  marketAvg5: number; marketAvg6: number;    // 全體均值 5日/6日（排除標的）
  sectorCode: string; sectorN: number; marketN: number;
  spread5: { market: number; sector: number };  // 差幅 = 標的 − 均值
  daily: Record<string, number>;             // 標的逐日%（供顯示/盤中疊加）
}
```

資料源（皆官方）：

| 用途 | 上市(TWSE) | 上櫃(TPEx) |
|---|---|---|
| 全市場逐日收盤 | `MI_INDEX` `response=json&date=YYYYMMDD&type=ALLBUT0999` → `tables[8].data`：代號(0)/收盤(8)/方向(9, 含 `green`=跌)/漲跌價差(10) | `afterTrading/dailyQuotes?date=YYYY/MM/DD&type=EW&response=json` → `tables[0].data`：代號(0)/收盤(2)/漲跌(3, 已帶 +/−) |
| 產業別 | `openapi.twse.com.tw/v1/opendata/t187ap03_L`：`公司代號`/`產業別` | `tpex.org.tw/openapi/v1/mopsfin_t187ap03_O`：`SecuritiesCompanyCode`/`SecuritiesIndustryCode` |

- 只取 4 碼純數字代號（排除 ETF/權證/債券等）。
- 上市方向欄：HTML 含 `green` → 跌(−)，否則漲(+)；上櫃漲跌欄已是帶號字串，直接 parse。
- trunc2：向零截斷至 2 位（比照既有 market-avg）。
- 兩市場各自成母體（上市標的比上市同類、上櫃比上櫃同類），不跨市場對齊產業碼。
- 快取 per `market:date`，6h（比照 `peratio` route）。

**上櫃同口徑定案**（使用者 2026-05-29 確認）：上櫃全體/同類均值與上市完全相同算法——上櫃每一檔個股累積漲幅**等權平均、排除標的本身**，不使用櫃買指數加權。舊筆記的「上櫃=櫃買指數」作廢。實作時仍會抓一檔上櫃處置股對拍 attstock 作回歸驗證，但口徑已鎖定為等權。

### 二、接進款一①②差幅判定（`DisposalTool.tsx` `thresh()` 與款一卡片）

目前 `thresh()` 僅 `max(價格門檻, 全體均值+20)`。改為三條並列：

- 價格門檻（款一① 上市32/上櫃30；款一② 上市25/上櫃23，且價差≥50元上市/40元上櫃）
- **全體差幅閘**：標的累積 − 全體均值 ≥ 20%
- **同類差幅閘**：標的累積 − 同類均值 ≥ 20%

UI（比照 attstock 截圖）顯示兩行：「全體均值 +X% 差幅 Y%（需≥20%）」「同類均值 +X% 差幅 Y%」。資料取自新 `sectoravg` API。

### 三、盤中即時卡片

- **收盤後**：卡片 0 = 下一交易日預測，當日(第6間隔)以 0% 計（現狀）。
- **交易日盤中**：同一張卡改為依今日市場即時計算，固定 **7 個更新時點**：10:00、11:00、12:00、12:30、13:00、13:10、13:20。盤中**只抓 7 次**（每個時點快取一份），並在 UI 明示「盤中於這 7 個時點更新」。
  - 標的即時價：MIS `getStockInfo.jsp?ex_ch=tse_<code>.tw`（帶 Referer header）。
  - 因一天僅 7 次，**同類/全體均值的當日項也逐檔即時、維持等權口徑一致**（不退用加權指數）。即每個時點抓一次全市場即時，重算等權當日%，疊進 6 日累積。
  - 收盤後該卡定案，生成下一張預測卡。
- 上櫃即時來源於實作時一併偵察（櫃買即時行情端點）。

### 四、移除非處置資料

- 移除殖利率/股利相關欄位與顯示（`peratio` route 已不回傳；確認 UI 無殘留）。
- PE/PBR 保留（款六），維持現有「假設達標」開關。

## 測試

- `sectoravg` 算法：以 2026/05/22~29 六日 fixture 對拍——國巨 5日=27.36、同類5日=6.15、全體5日=2.22、同類6日=8.95（黃金值，鎖死回歸）。
- trunc2 / 方向解析 / 6日窗口交集 單元測試。
- 上櫃：抓一檔上櫃處置股對拍 attstock 後補黃金值測試。
- 款一①② 差幅閘：合成標的，驗證全體與同類兩條閘各自觸發/不觸發。
- 盤中卡片：mock 7 時點即時，驗證疊加與快取。

## 風險 / 未決

1. **盤中即時全市場抓取**——7 次/日可接受；上櫃即時端點待偵察。
2. attstock 是否排除「已注意股」——舊筆記提到上市全體略低疑似排除注意股；目前等權排除標的本身已對上 2.22，暫不額外排除，若日後對不上再加。

## 範圍外（不做）

- 跨市場同類合併、加權類股指數法、殖利率/股利、attstock 即時依賴。
